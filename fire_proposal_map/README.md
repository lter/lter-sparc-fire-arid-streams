# NSF Fire-Stream Figure Workflow

This project builds a publication-quality map figure for an NSF proposal on
wildfire effects in aquatic systems.

The figure includes:
- MTBS fire perimeters intersecting the study catchment polygon
- Higher-order NHDPlus stream flowlines (Strahler order >= 5)
- Major city points and labels
- Major river labels from NHD GNIS names
- A cached hillshade + state-boundary basemap
- A CONUS inset locating the study region
- A north arrow, scale bar, and legends

## Project Structure

- `build_nsf_fire_figure.R`: main end-to-end figure build script
- `download_hydrology_cache.R`: prefetches and caches NHDPlus flowlines
- `download_basemap_cache.R`: prefetches and caches basemap assets
- `MAP_EDIT_PROMPT.md`: reusable prompt template for future map edit sessions
- `PROJECT_LOG.md`: living log of goals, workflow, tuning knobs, and change history
- `catch.geojson`: study catchment geometry
- `mtbs_perims_DD.shp` (+ sidecar files): MTBS fire perimeter source
  (local-only, not committed)

```sh
ogr2ogr -f GeoJSON /tmp/catch.geojson "PG:host=localhost user=srearl
dbname=wildfire active_schema=firearea" -dialect PostgreSQL -sql "SELECT
ST_Union(geometry) AS geometry FROM firearea.ecoregion_catchments"
```

- `data_cache/`: cached hydrology and basemap artifacts
- `outputs/`: final figure and diagnostics outputs

## Requirements

- R 4.0+
- Internet access for initial cache generation
- Write permissions in the project directory

The scripts install missing R packages automatically from CRAN.

## Quick Start

From the project root:

```bash
Rscript download_hydrology_cache.R
Rscript download_basemap_cache.R
Rscript build_nsf_fire_figure.R
```

This creates all cache artifacts and then writes the final figure.

### Local MTBS Requirement

MTBS perimeter files are intentionally not version-controlled due to file size.
The figure builder searches for MTBS data in this order:

1. `./mtbs_perims_DD.shp`
2. `/home/srearl/Desktop/fire_proposal_map/mtbs_perims_DD.shp`
3. `MTBS_SHP_PATH` (environment variable)

Example:

```bash
MTBS_SHP_PATH=/path/to/mtbs_perims_DD.shp Rscript build_nsf_fire_figure.R
```

## Caching

Caching is used to speed up repeated cartographic edits.

### Hydrology Cache

Generated by `download_hydrology_cache.R` or on-demand by
`build_nsf_fire_figure.R`.

Typical files:
- `data_cache/nhdplus_flowlines_so5_bbox.gpkg`
- `data_cache/nhdplus_flowlines_so5_bbox_meta.csv`

### Basemap Cache

Generated by `download_basemap_cache.R` or on-demand by
`build_nsf_fire_figure.R`.

Typical files:
- `data_cache/conus_states_2024_epsg5070.gpkg`
- `data_cache/hillshade_bbox_epsg5070.tif`
- `data_cache/basemap_cache_meta.csv`

## Refresh Controls

Use environment variables to force cache refresh:

- `REFRESH_HYDRO_CACHE=1`
- `REFRESH_BASEMAP_CACHE=1`

Examples:

```bash
REFRESH_HYDRO_CACHE=1 Rscript download_hydrology_cache.R
REFRESH_BASEMAP_CACHE=1 Rscript download_basemap_cache.R
REFRESH_HYDRO_CACHE=1 REFRESH_BASEMAP_CACHE=1 Rscript build_nsf_fire_figure.R
```

## Cartographic Notes

- Main map projection: EPSG:5070 (NAD83 / Conus Albers)
- Stream plotting threshold: Strahler order >= 5
- Layer clipping: northern map extent is clipped at 38°N by default
- Fire symbology: two-color gradient by fire decade derived from `Ig_Date`
- Major cities: top cities (population >= 100,000) inside map extent
- Major rivers: top named rivers by cumulative length (`gnis_name`)
- Inset placement: top-right of the main figure

To change the latitude clip:

```bash
MAP_CLIP_LAT=39 Rscript build_nsf_fire_figure.R
```

## Troubleshooting

### Empty or Missing Streams

- Run with a forced hydrology refresh:

```bash
REFRESH_HYDRO_CACHE=1 Rscript download_hydrology_cache.R
```

### Basemap/Hillshade Issues

- Force a basemap refresh:

```bash
REFRESH_BASEMAP_CACHE=1 Rscript download_basemap_cache.R
```

### Clean Rebuild

If you want to fully rebuild caches and outputs:

```bash
rm -rf data_cache outputs
Rscript download_hydrology_cache.R
Rscript download_basemap_cache.R
Rscript build_nsf_fire_figure.R
```

## Reproducibility Suggestions

- Keep the generated `*_meta.csv` files with proposal materials.
- Record R session details when producing final deliverables.
- Commit script and cache configuration changes together.
